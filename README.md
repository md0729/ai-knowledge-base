# 🧠 AI Knowledge Base

This repository serves as a structured knowledge system for AI, computer vision, and system integration projects.
The goal is to transform scattered learning and project experience into reusable engineering knowledge.

---

## 🎯 Purpose

Instead of using AI tools for one-time answers, this repository is designed to:

* Accumulate knowledge over time
* Organize technical concepts systematically
* Record real project experience and debugging insights
* Support project development, thesis writing, and interviews

---

## 🧱 Repository Structure

```
/wiki_core      → Core knowledge (concepts, models, system design)
/projects       → Real project experience (debug, issues, solutions)
/raw            → Unprocessed inputs (articles, ideas, notes)
```

---

## 📌 Knowledge Classification Rules

To keep the system organized, all content follows this rule:

### 🧠 wiki_core

* Reusable knowledge
* Abstract concepts
* System design and methodology
* Can be applied across different projects

Examples:

* LSTM window size design
* YOLO + LSTM multi-modal fusion
* Human-robot interaction pipeline

---

### 🔧 projects

* Real implementation experience
* Debugging records
* System integration issues
* Problem → Analysis → Solution

Examples:

* Socket disconnect issue
* MediaPipe unstable detection
* Model integration challenges

---

### 🗑 raw

* Unprocessed inputs
* Articles, ideas, or quick notes
* No structure required

---

## 🔄 Workflow

```
Input (ideas / problems / articles)
→ Store in raw (optional)
→ Use AI to structure into wiki format
→ Save into wiki_core or projects
```

---

## 🧠 Core Concept

This is not just a note-taking repository.

It is a **growing knowledge system** where:

* Knowledge is structured
* Experience is preserved
* AI is used as a tool for organization

---

## 🚀 Future Direction

* Expand knowledge coverage
* Improve classification and structure
* Introduce semi-automation for knowledge organization
* Support system design and research development

---

## 💡 Key Principle

> If the knowledge can be reused → put it in `wiki_core`
> If it comes from real experience → put it in `projects`
